# Observability Stack

## Overview
This repository documents observability practices used to ensure platform
reliability through meaningful metrics, actionable alerts, and service-level
objectives (SLOs).

The focus is on **signal over noise** and **operational relevance**.

## Scope
- Metrics and golden signals
- Alerting philosophy and examples
- SLO and error budget definitions
- Incident and escalation linkage

## Non-Goals
- Dashboard aesthetics
- Tool-specific UI configuration
- Alerting on every metric

## Relationship to Incident Response
Alerts defined here are designed to trigger incident response workflows
documented in the `incident-response-sre` repository.
